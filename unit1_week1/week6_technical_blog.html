Functional and Object Oriented programming(OOP) are two programming paradigms used by Developers. In Object oriented programming we define Classes which will have certain methods. We create objects as a Instance of the class and then use the Class Methods to peform coding tasks. Each object has its own attributes. These attributes can be modifed and the Class methods will then use the new object attributes. Functional programming on the other hand relies on user input to fetch Data or perform actions. Every time we need to use the function we need to pass in the function name and optional arguments and get the Data we are looking for.

Here is an example of OOP vs Functional programming:

Supposed we want to store Data for a group of Students attending a Ruby Class:

OOP:

class StudentData
def initialize(name,age,gender)
@name = name
@age = age
@gender = gender
end
attr_writer :name,:age,:gender
def data
puts "Name: #{@name}\nAge: #{@age}\nGender: #{@gender}"
end
end

We can now create object like:

adam = StudentData.new("Adam","20","M")
sara = StudentData.new("Sara","21","F")

And use the Class Method to get Data:
e.g: adam.data which gives us:

Name: Adam
Age: 20
Gender: M

Or sara.data which will give us:

Name: Sara
Age: 21
Gender: F

With Object oriented, we can also change the Object attributes. If we want to change the age for adam object, we can do
adam.age = 22 and now on calling adam.data will give us:

Name: Adam
Age: 22
Gender: M

With Functional programming we will have a function like:

def student_data(name,age,gender)
puts "Name: #{name}\nAge: #{age}\nGender: #{gender}"
end

We will now have to call the function like: student_data("Adam",20,"M") to get the Data we want. If we have to change any attribute we will have to call the function again with different set of attributes.

As you see from the simple example above thereare numerous advantages of using OOP vs Functional programming.
