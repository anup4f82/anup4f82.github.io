Blog: Regular Expressions

A Regular Expression is a sequence of characters that helps you match or find strings or sets of strings using specialized syntax held in a pattern. Regular Expressions are used to determine if a string contains a given pattern or extract portions that match. This is very useful tool to search through huge amounts of Data like Files,Databases etc.

A Regular Expression is delimited with forward slashes (/).

Below I have provided some simple examples of how Regular expressions are used to find string patterns in Ruby

[abc] - A single character of a,b,c

/[abcd]/.match 'ruby'
Output: #<MatchData "b" >
/[xyzn]/.match 'ruby'
Output: nil

The match method above returns a match object. In the first example above, the regular expression returns a Match object since 'b' is present in 'ruby'. In the second example none of the characters in [xyzn] exist in 'ruby' and hence it returns nil

[^abc] - Any character except a,b or c

/[^ruby]/.match 'ruby'
Output: nil
/[^abc]/.match 'ruby'
Output: #<MatchData "r">

In the first example above, the Output is nil, since the string does not have any characters except 'r','u','b','y'. In the second example, it returns a object since 'r' is not one of 'a','b' or 'c'.

[a-z] - Any character in range of a-z

/[x-z]/.match 'ruby'
Output: #<MatchData "y">
/[c-q]/.match 'ruby'
Output: nil

In the first example above, it returns the character 'y' since it finds y in the range provided : [x-z]. In the second example the Output is nil since it doesn't find 'r','u','b','y' in the range [c-q].

^ - Start of line

/^b/.match 'ruby'
Output: nil/
/^r/.match 'ruby'
Output: #<MatchData "r">

In the first example since 'b' is not the first character in string 'ruby' it returns nil. In the second example it returns an object since 'r' is the first character in 'ruby'.
